# Supabase Auth Hooks Setup Guide

This guide explains how to configure Supabase Auth Hooks to use your deployed AuthKey-based OTP API for SMS delivery.

## Overview

With Supabase Auth Hooks, your app code remains unchanged. When users call `supabase.auth.signInWithOtp({ phone })`, Supabase automatically triggers your custom OTP endpoint instead of using its default SMS provider.

## Setup Steps

### 1. Enable Supabase Auth Hooks (Beta)

1. Go to your Supabase Dashboard
2. Navigate to **Authentication** → **Hooks**
3. Enable the **OTP Send Hook** (Beta feature)

### 2. Configure the OTP Send Hook

⚠️ **IMPORTANT**: Double-check these settings to avoid 404 errors:

- **Endpoint URL**: `https://authkey-otp-api-production.up.railway.app/send-otp`
  - ✅ Verify URL is exactly correct (no extra spaces/characters)
  - ✅ Must end with `/send-otp`
- **HTTP Method**: `POST` (Supabase sends POST requests)
- **Secret Header**: 
  - Key: `secret` (case-sensitive, lowercase)
  - Value: Your actual secret key from Railway API
  - ❌ Don't use the webhook format (`v1,whsec_...`)
  - ✅ Use the plain secret key only

### 3. Hook Request Format

When a user requests an OTP, Supabase will send a POST request to your endpoint:

```
POST https://authkey-otp-api-production.up.railway.app/send-otp
Content-Type: application/json
Secret: {your_secret}

{
  "phone": "+1234567890",
  "token": "123456",
  "user": {
    "id": "user-uuid",
    "phone": "+1234567890"
  }
}
```

Where:
- `phone`: The user's phone number
- `token`: The OTP code generated by Supabase
- `user`: User object with additional details
- Secret is sent in the header as configured

### 4. Expected Response

Your endpoint should return:
- **Success**: HTTP 200 with JSON response
- **Error**: HTTP 4xx/5xx with error details

### 5. App Code (No Changes Required)

Your existing app code continues to work as-is:

```javascript
// This will now use your AuthKey API via the hook
const { error } = await supabase.auth.signInWithOtp({
  phone: formattedPhoneNumber,
});
```

## Flow Diagram

```
App calls supabase.auth.signInWithOtp()
        ↓
Supabase generates OTP code
        ↓
Supabase triggers your hook endpoint
        ↓
Your Railway API calls AuthKey
        ↓
AuthKey sends SMS to user
        ↓
User receives OTP
        ↓
User enters OTP in app
        ↓
Supabase verifies OTP
```

## Environment Variables

Update your `.env` file with:

```env
# These are for reference only - actual configuration is done in Supabase Dashboard
AUTH_HOOK_OTP_ENDPOINT=https://authkey-otp-api-production.up.railway.app/send-otp
AUTH_HOOK_SECRET=your-auth-hook-secret-key
```

## Testing

1. Ensure your Railway API is running and accessible
2. Test the hook endpoint directly:
   ```bash
   curl "https://authkey-otp-api-production.up.railway.app/send-otp?phone=+1234567890&token=123456&secret=your_secret"
   ```
3. Test OTP flow in your app
4. Check Supabase logs for hook execution status

## Troubleshooting

### Common Issues:

1. **Hook not triggering**: Verify the hook is enabled and endpoint URL is correct
2. **403/401 errors**: Check the secret header configuration
3. **Timeout errors**: Ensure your Railway API responds within 10 seconds
4. **OTP not received**: Verify AuthKey API credentials and phone number format

### Debug Steps:

1. Check Supabase Auth logs
2. Monitor Railway API logs
3. Verify AuthKey API status
4. Test with different phone numbers

## Security Notes

- Keep your secret key secure and never expose it in client-side code
- Use HTTPS for all endpoints
- Implement rate limiting on your OTP endpoint
- Validate phone number formats
- Log failed attempts for monitoring
